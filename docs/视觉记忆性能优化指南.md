# 视觉记忆性能优化指南

本文档说明如何在 LycheeMem-bendi 中使用极速版视觉记忆组件，大幅提高视觉识别速度。

## 🚀 性能提升对比

| 组件 | 标准版 | 极速版 | 提升 |
|------|--------|--------|------|
| 图片压缩 | 1024px, quality=85 | 512px, quality=75 | **2-3x** |
| VLM 超时 | 30 秒 | 15 秒 | **2x** |
| Prompt 长度 | ~150 字 | ~30 字 | **3-5x** |
| 模型选择 | qwen-vl-max | qwen-vl-max-lite | **2-3x** |
| 嵌入精度 | FP32 | FP16 (GPU) | **1.5-2x** |
| 缓存大小 | 128 条 | 256 条 | **命中率 +50%** |

**总体性能提升：3-10 倍**（取决于图片和 API 响应速度）

## 📦 极速版组件

### 1. VisualExtractorFast（极速版图片理解）

**优化特性：**
- ✅ 超轻量 Prompt（仅 30 字核心指令）
- ✅ 激进图片压缩（512px，quality=75）
- ✅ 短超时（15 秒）
- ✅ 大缓存（256 条）
- ✅ 低 temperature（0.01）
- ✅ 少 token（400-600）

**使用方法：**

```python
from src.memory.visual import VisualExtractorFast

# 创建极速版提取器
extractor = VisualExtractorFast(
    llm=llm_adapter,
    fast_mode=True,           # 快速模式
    ultra_fast_mode=True,     # 超快速模式（可选）
    max_image_size=512,       # 图片最大边长
    image_quality=75,         # JPEG 压缩质量
    cache_size=256,           # LRU 缓存大小
    timeout=15,               # VLM 调用超时（秒）
)

# 使用（与标准版 API 相同）
result = await extractor.extract_from_base64(image_b64, session_id="session_001")
```

### 2. MultimodalEmbedderFast（极速版多模态嵌入）

**优化特性：**
- ✅ 小模型优先（clip-vit-base-patch32，512 维）
- ✅ 批量处理（batch_size=64）
- ✅ 大缓存（2000 条）
- ✅ FP16 半精度推理（GPU 加速）
- ✅ 异步非阻塞 I/O

**使用方法：**

```python
from src.memory.visual import MultimodalEmbedderFast

# 创建极速版嵌入器
embedder = MultimodalEmbedderFast(
    model_name="openai/clip-vit-base-patch32",  # 最快的模型
    device=None,              # 自动选择 (GPU 优先)
    lazy_load=True,           # 延迟加载
    cache_size=2000,          # LRU 缓存大小
    use_fp16=True,            # FP16 半精度（GPU 加速）
    batch_size=64,            # 批量处理大小
    use_dashscope=False,      # 是否使用阿里云 API
)

# 文本嵌入
text_embedding = embedder.embed_text("红色图片")

# 图像嵌入
image_embedding = embedder.embed_image("path/to/image.jpg")

# 批量嵌入（更高效）
text_embeddings = embedder.batch_embed_texts(["文本 1", "文本 2", ...])
image_embeddings = embedder.batch_embed_images([image1, image2, ...])
```

## 🔧 配置示例

### 在 main.py 中使用极速版组件

```python
from src.memory.visual import VisualExtractorFast, MultimodalEmbedderFast
from src.memory.visual.visual_store import VisualStore
from src.memory.visual.visual_retriever import VisualRetriever

# 1. 创建极速版多模态嵌入器
multimodal_embedder = MultimodalEmbedderFast(
    model_name="openai/clip-vit-base-patch32",
    use_fp16=True,
    cache_size=2000,
    batch_size=64,
)

# 2. 创建视觉存储（使用极速版嵌入器）
visual_store = VisualStore(
    db_path="data/visual_memory.db",
    vector_db_path="data/visual_vector",
    image_storage_path="data/visual_images",
    multimodal_embedder=multimodal_embedder,  # 传入极速版
)

# 3. 创建极速版图片提取器
visual_extractor = VisualExtractorFast(
    llm=llm_adapter,  # 建议使用 qwen-vl-max-lite 等轻量模型
    fast_mode=True,
    ultra_fast_mode=True,
    max_image_size=512,
    image_quality=75,
    cache_size=256,
    timeout=15,
)

# 4. 创建检索器
visual_retriever = VisualRetriever(visual_store=visual_store)
```

## ⚠️ 注意事项

### 1. 质量 vs 速度权衡

极速版会略微降低识别质量以换取速度：
- 图片压缩可能导致细节丢失
- 短 Prompt 可能减少提取的结构化信息
- 小模型可能降低嵌入质量

**建议：**
- 对速度敏感场景使用极速版（实时对话、批量处理）
- 对质量敏感场景使用标准版（重要文档、精细分析）

### 2. GPU 要求

FP16 半精度推理需要 GPU 支持：
- NVIDIA GPU：需要 compute capability >= 7.0（Volta 架构及以上）
- CPU 模式下自动降级为 FP32

### 3. 内存使用

更大的缓存会占用更多内存：
- VisualExtractorFast: ~50MB（256 条缓存）
- MultimodalEmbedderFast: ~200MB（2000 条缓存）

### 4. 模型依赖

需要安装额外的依赖：

```bash
# 安装 sentence-transformers（本地嵌入）
pip install sentence-transformers

# 安装 Pillow（图片处理）
pip install Pillow

# 安装 DashScope（阿里云 API，可选）
pip install dashscope
```

## 🔍 故障排查

### 问题 1：FP16 不支持

```
RuntimeError: torch.float16 is not supported on CPU
```

**解决：** FP16 仅在 GPU 上支持，CPU 模式下设置 `use_fp16=False`。

### 问题 2：模型下载失败

```
OSError: Can't load model 'openai/clip-vit-base-patch32'
```

**解决：**
1. 检查网络连接
2. 使用镜像：`export HF_ENDPOINT=https://hf-mirror.com`
3. 手动下载模型到本地

### 问题 3：缓存占用过高

```
Warning: Cache size exceeded limit
```

**解决：** 减小 `cache_size` 参数或定期调用 `flush()` 方法。
